# Performance optimizations (Lighthouse / PageSpeed)

## Summary

Optimizations target **Performance 95+**, **FCP <2s**, **LCP <2.2s**, **Speed Index <3.5s** while keeping design and accessibility (contrast fix).

---

## 1. Modified HTML head (all pages)

- **Preconnect:** `fonts.googleapis.com` and `fonts.gstatic.com` (crossorigin) to speed up font requests.
- **Preload hero (homepage only):** `rel="preload" as="image"` for hero 960w image (~60KB) so LCP image starts early.
- **Preload CSS:** `rel="preload" href="/assets/css/styles.min.css" as="style"`.
- **Non-blocking fonts:** Google Fonts stylesheet with `media="print" onload="this.media='all'"` so it doesn’t block first paint.
- **Non-blocking main CSS:** `styles.min.css` loaded with `media="print" onload="this.media='all'"`.
- **Noscript fallback:** `<noscript><link rel="stylesheet" href="/assets/css/styles.min.css" /></noscript>`.
- **Critical CSS (homepage only):** Inline `<style>` with above-the-fold rules (layout, nav, hero, kicker, h1, lead, btns, hero-trust) so first paint doesn’t wait on the full stylesheet.

---

## 2. Hero image LCP (homepage)

- **Preload:** `rel="preload" as="image"` for `Resized_1_2_...-960w.webp` (~60KB, under 100KB).
- **Responsive:** `srcset` with 320w, 640w, 960w, 1280w variants; `sizes="100vw"` for hero bg.
- **Attributes:** `fetchpriority="high"`, `decoding="async"`, no `loading="eager"` (preload handles priority).
- **Default src:** 960w variant as fallback and for preload match.

---

## 3. Responsive images (all pages)

For each image in `/assets/images/`:

- **Variants:** 320w, 640w, 960w, 1280w generated by `scripts/optimize-images.sh` (cwebp -q 78, hero 75).
- **Markup:** `src` points to 640w (or 960w for hero); `srcset` lists all four; `sizes` set by context (e.g. `(max-width:900px) 100vw, 50vw` for side-by-side content).
- **Lazy load:** `loading="lazy"` for below-fold images; hero bg has no loading attribute (and preload).
- **Decoding:** `decoding="async"` on all images.

---

## 4. Image compression

- **Script:** `scripts/optimize-images.sh` generates responsive WebP with cwebp (quality 78; hero 75).
- **Hero 960w:** Kept under 100KB (current ~60KB); if over 100KB script re-encodes at q72.
- **Gallery/section images:** Served as responsive variants; 640w/960w typically well under 80KB.

---

## 5. Fonts

- **Preconnect:** In head (see §1).
- **display=swap:** Google Fonts URL includes `&display=swap` so text shows immediately with system fallback.
- **No @import in CSS:** Fonts loaded via `<link>` in HTML so main CSS doesn’t block on font CSS.

---

## 6. CSS minification

- **Tool:** `cssnano-cli` (run via `scripts/minify-css.sh`).
- **Input:** `public/assets/css/styles.css` (~19KB).
- **Output:** `public/assets/css/styles.min.css` (~15KB).
- **Serving:** All pages reference `styles.min.css`; cache headers in nginx (30d, immutable for static assets).

---

## 7. Forced reflow / JS

- **Before/after slider:** Update handler wrapped in `requestAnimationFrame` so `wrap.style.width` is applied in the next frame instead of causing synchronous layout.
- **IntersectionObserver:** Used for reveal; no layout reads after DOM writes in the same frame.
- **Script:** Already `defer`; no extra layout-thrashing patterns (no offsetWidth/clientHeight/getBoundingClientRect in hot paths).

---

## 8. Accessibility contrast

- **Kicker:** Color changed from `var(--crimson)` (#a30d15) to `#e85c63` so contrast on `--ink` (#08090b) meets WCAG AA (≥4.5:1).
- **Location:** `public/assets/css/styles.css` (and reflected in inline critical CSS on homepage).

---

## Optional (already in place)

- **Cache:** nginx `location ~* \.(css|js|...|webp)$` sets `Cache-Control: public, max-age=2592000, immutable`.
- **Lazy loading:** Below-fold images use `loading="lazy"`.
- **Defer JS:** `<script defer src="/assets/js/main.js">`.

## Before vs after (expected)

| Metric | Before | After |
|--------|--------|--------|
| Render-blocking CSS | styles.css blocked first paint | Critical inline; full CSS async → first paint sooner |
| LCP | Hero image 133KB, single size, eager | Hero preloaded 960w ~60KB; srcset; decoding=async → faster LCP |
| Image payload | Full-size images | Responsive srcset → smaller bytes per viewport |
| Fonts | @import in CSS blocked parse | Preconnect + async link with display=swap → no block |
| CSS size | ~19KB | ~15KB minified |
| Layout thrashing | Slider wrote then potential read | rAF batch → less reflow risk |
| Contrast | Kicker below 4.5:1 | #e85c63 → pass |
